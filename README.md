Este protótipo foi desenvolvido para **estudantes universitários** que dividem apartamento ou moram em repúblicas e precisam organizar despesas coletivas como aluguel, contas e mercado.



Heurística de Nielsen aplicada

Visibilidade do status do sistema

O sistema foi projetado para deixar sempre claro:

* Quem deve
* Quanto deve
* Para quem deve

Isso reduz confusão e melhora a transparência na divisão de gastos.
 Telas do Protótipo

1. Dashboard de Grupos

   * Exibe todos os grupos ativos
   * Mostra saldo geral (quem deve ou recebe)

2. Detalhes do Grupo

   * Lista de gastos do grupo
   * Identificação de quem pagou

3. Adicionar Gasto

   * Inserção de valor, descrição e participantes
   * Escolha do tipo de divisão

4. Câmera / Scanner

   * Simulação de leitura de nota fiscal

5. Extrato de Dívidas

   * Exibe quem deve para quem
   * Botão para realizar pagamento

6. Confirmação de Pagamento

   * Feedback visual após pagamento realizado

Fluxo de Tarefa

Cadastrar e dividir uma conta de luz entre 3 pessoas:

1. O usuário acessa o Dashboard
2. Seleciona o grupo "Casa"
3. Clica em "Adicionar Gasto"
4. Insere o valor da conta de luz
5. Seleciona os 3 participantes
6. Escolhe divisão igual
7. Confirma o cadastro

Decisões de Design

* Interface simples e direta (baixa fidelidade)
* Foco em informações financeiras importantes
* Uso de hierarquia visual para destacar valores
* Navegação clara com setas indicando o fluxo

Observação

Este é um protótipo de baixa fidelidade com foco em usabilidade e clareza, não em design visual final.
